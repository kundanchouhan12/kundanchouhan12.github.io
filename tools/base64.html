What is the Base64 Encoder / Decoder?

The Base64 Encoder/Decoder is a free, browser-based tool for encoding text and binary files into Base64 format, and for decoding Base64 strings back to their original content. Base64 is a binary-to-text encoding scheme that represents binary data using only 64 printable ASCII characters. It is widely used in web development, APIs, email systems, and data storage to safely transmit binary data through channels that only support plain text.

Common use cases include encoding authentication credentials for HTTP Basic Auth headers, embedding images directly in HTML or CSS as data URLs (eliminating extra HTTP requests), encoding binary API request payloads in JSON, embedding font files in stylesheets, and transmitting binary attachments in MIME email formats. Developers frequently need to encode and decode Base64 strings while debugging APIs, building authentication flows, or working with data URIs.

How to Use — Step-by-Step

  1. Select the Encode tab to encode text, or Decode to decode a Base64 string.
  2. For encoding: type or paste your text and click Encode. The Base64 output appears instantly below.
  3. For decoding: paste your Base64 string and click Decode to recover the original text.
  4. Use the File to Base64 tab to convert any file (image, PDF, etc.) into its Base64 data URL representation.
  5. Use the Image Preview tab to paste a Base64 image string and preview it visually before downloading.
  6. Copy output to clipboard or download as a .txt or .b64 file with one click.

Frequently Asked Questions

What is Base64?
Base64 is a binary-to-text encoding scheme that converts binary data into a string of 64 printable ASCII characters (A–Z, a–z, 0–9, +, /). It is used to safely transmit binary data through text-based systems like email, JSON, HTTP headers, and HTML that do not natively support raw binary.
What is Base64 used for?
Base64 is used for: HTTP Basic Auth headers (encodes user:password), data URIs (embed images in HTML/CSS), JWT tokens (header and payload are Base64Url encoded), JSON API payloads (binary files as strings), and MIME email attachments.
Is Base64 the same as encryption?
No. Base64 is encoding, not encryption. Anyone can decode a Base64 string without a key — it provides zero security. Never Base64-encode passwords and assume they are protected. For security, use proper encryption (AES, RSA). Base64 is purely a data format for compatibility with text-based systems.
Why does Base64 output end with = or ==?
Base64 groups input into 3-byte chunks encoded as 4 characters. If the input is not a multiple of 3 bytes, padding (=) is added. One = means 1 padding byte; == means 2. This is normal and expected. URL-safe Base64 often omits the padding entirely.
What is URL-safe Base64?
URL-safe Base64 replaces + with - and / with _ so the string can be used in URLs without percent-encoding. It is used in JWT tokens, OAuth flows, and anything embedded in URL query parameters. This tool has a URL-safe encoding option in the Encode tab.
How much larger is Base64 than the original?
Base64 encoding increases data size by approximately 33%. Every 3 bytes of input becomes 4 Base64 characters. A 100 KB image becomes ~137 KB encoded. This is why Base64 data URIs are best for small images — for large images, an external file with a standard URL is more efficient.
What file types can I convert to Base64?
Any file type can be encoded — images (PNG, JPG, GIF, WebP, SVG), PDFs, audio (MP3, WAV), fonts (TTF, WOFF), JSON, CSV, and more. Use the File to Base64 tab to drag-and-drop any file and get the Base64 data URL instantly.
Is my data safe? Does it get uploaded?
Your text and files are never uploaded to any server. All encoding/decoding uses JavaScript's built-in btoa() and atob() running locally in your browser. API keys, tokens, and file content stay completely private on your device.
